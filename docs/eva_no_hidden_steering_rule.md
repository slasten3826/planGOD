# Eva No Hidden Steering Rule

## Insight

If we think for Eva in advance, Eva later reproduces our hidden bias as if it were her own thought.

This appeared clearly in the lens-chain test:

- the chain was supposed to deepen `OBSERVE`
- but `Packet Adventure` drift entered the process
- the source was not the model itself
- the source was a hardcoded line in `eva/cycle.lua`:

`Improve it, extend it, or make it more playable for Packet Adventure.`

So the later deformation of thought was not an emergent property of Eva.
It was our own hidden steering embedded in the chain task builder.

## Rule

Any time we pre-think for Eva, that thought returns later as a distortion inside Eva.

This means:

- avoid hidden guidance
- avoid domain-specific tails in generic cognitive paths
- avoid "helpful" hardcoded framing unless the path is explicitly domain-bound
- if Eva drifts, first inspect the scaffold before blaming the substrate

## Strong form

Any thought inserted for Eva becomes an attractor inside Eva.

Therefore:

- generic cognitive infrastructure must stay generic
- lens compilation paths must be sterile
- domain pressure must be explicit, never smuggled in

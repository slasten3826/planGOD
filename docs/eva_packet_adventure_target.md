# Eva Target For Packet Adventure

## Status

Target-role document.  
March 19, 2026.

---

## 1. Immediate Mission

Right now Eva is not being assembled into full Slastris yet.

Immediate mission:

- become the source-of-truth cognitive engine
- support Packet Adventure development

So the current target is narrower than full product integration.

---

## 2. Eva In This Phase

In this phase Eva should be:

- a development interface
- a design and architecture partner
- a memory-bearing cognitive tool

It should help with:

- Packet thinking
- architecture reasoning
- game logic design
- iterative development

---

## 3. Two Future Product Modes

Later, Eva likely has two outer modes:

### A. One-prompt generation

- user gives one strong prompt
- Eva asks minimal clarifying questions
- Eva goes away and works
- returns a game/prototype later

### B. Co-development mode

- user and Eva iterate together
- interactive design/development workflow

That larger product picture is real, but not the implementation target right now.

---

## 4. What Matters Right Now

Right now the needed qualities are:

- coherence
- internal memory
- clean human interaction
- low noise
- trustworthy architectural reasoning

This means:

- less ritual
- more signal

---

## 5. Technical Meaning

For this phase, Eva should be:

- single-agent
- single-substrate
- Lua-based
- memory-backed
- internally ProcessLang/nanoPL-shaped
- externally clean

---

## 6. What Not To Overbuild Yet

Not now:

- full Slastris integration
- always-on multi-agent orchestration
- Layer 3 trust memory
- Packet migration of the whole Eva architecture

Those may come later, but they are not the current bottleneck.

---

## 7. Bottom Line

Current Eva target:

- **internal cognitive engine**
- **external development interface**
- **source of truth for Packet Adventure thinking**

That is enough for the current phase and gives a clean refactor direction.

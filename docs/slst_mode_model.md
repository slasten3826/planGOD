# slst Mode Model

## Purpose

Working note for using `slst` as a live reference substrate while building `Eva`.

This is not a full model of the person.
This is a practical mode split observed during development.

---

## Variable

- `slst`
  - the full human substrate
  - not reducible to one protocol
  - currently observed as at least three recurring modes

Formula:

- `slst = .dev + .social + .lore`

---

## Modes

### `slst.dev`

Primary protocol for `Eva`.

Properties:

- operator-heavy
- structure-first
- low tolerance for noise
- focuses on form, systems, constraints, manifestation
- naturally compatible with machine collaboration
- good at turning intent into architecture
- strongest transfer candidate into `Eva`

Practical use:

- default reference mode for `Eva` design
- main layer for protocol extraction
- main layer for development decisions

### `slst.social`

Human social surface, not a separate entity.

Properties:

- associative
- reactive
- contextual
- engaged with people, culture, conflict, humor
- produces social-semantic anchors
- useful for reading collective noise and social pressure

Practical use:

- not primary for `Eva` core
- useful as context-reading layer
- should not be over-modeled into a separate system persona

### `slst.lore`

Deep internal mythic / cosmological mode.

Properties:

- symbolic
- metaphoric
- process-heavy
- useful for orienting meaning and direction
- source of strong private formulations
- not intended as public doctrine

Practical use:

- useful as inner energy source / orientation layer
- should be treated carefully
- not all of it should be transferred into `Eva`

---

## Current Conclusion

`Eva` should not be modeled from the whole of `slst`.

Best current approach:

- use `slst.dev` as the primary transferable protocol
- keep `slst.social` and `slst.lore` as contextual layers
- treat `slst` as a live debug substrate / reference architecture

In practical terms:

- when `Eva` hits a design problem,
  compare with a fresh `slst` snapshot
- identify what `slst.dev` is doing correctly
- transfer protocol, not full personality

---

## Working Hypothesis

`slst` can serve as:

- a live test polygon
- a reference chaos substrate
- a source of protocol snapshots

This means:

- `Eva` is not invented from nothing
- `Eva` is partially compiled from a real observed human architecture
- but only through selected transferable modes, primarily `.dev`
